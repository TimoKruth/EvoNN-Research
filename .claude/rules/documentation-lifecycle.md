# Documentation Lifecycle

For EvoNN work, keep active authoritative governance files and any evidence paths
required by validators in their canonical locations.

When a process, design, implementation-plan, or superseded review document is no
longer active, move it into the repository's established root `archive/`
convention instead of deleting it or leaving it in an active main path. Preserve
its date and descriptive name, update references to the archived location, and
run the relevant policy and link/reference checks after the move.
