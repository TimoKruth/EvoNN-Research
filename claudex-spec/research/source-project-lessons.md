# Source Project Lessons

This document records how the predecessor ecosystem informed the clean-slate specification. It is non-normative except where linked to requirements.

## Preserve

- Multiple search abstractions rather than one premature merged kernel.
- Family-first, topology-first, hierarchy-first, and primitive-first research roles.
- Canonical benchmark IDs with explicit native mappings.
- Strong non-evolutionary contenders.
- Per-run DuckDB for analytical evidence and parallel-run safety.
- Content-addressed candidates where practical.
- Explicit failure rows and visible unsupported/skipped states.
- Multi-fidelity evaluation and promotion.
- Specialist, Pareto, quality-diversity, and deployment-aware views.
- Artifact-based engine interoperability.
- Read-only evidence inspection concepts.
- Budget, telemetry, evidence promotion, and advancement gates.
- Local Apple Silicon truth path and explicit portability qualification.

## Fix

- Repository and terminology drift.
- Multiple conflicting pack registries and legacy system IDs.
- Unknown config fields silently ignored.
- Raw heterogeneous metric aggregation.
- Different meanings of evaluation count and epochs.
- Incomplete fairness checks.
- Unseeded MLX/NumPy/worker randomness.
- Resume that does not reproduce uninterrupted execution.
- Non-atomic checkpoints and ambiguous checkpoint phase.
- Export that retrains or fabricates missing results.
- Multiple run IDs for one logical run.
- Writable inspectors/exporters.
- Absolute paths in portable artifacts.
- Ad hoc database migrations.
- Incorrect parameter/model-byte proxies labeled as measurements.
- NEAT claims without homologous innovations or active speciation.
- Stale archive indices and incomplete genotype persistence.
- Hierarchy/QD implementation without decision-grade evidence.
- Dashboard status inferred from files/logs instead of explicit state.
- Unsafe URL/archive/model acquisition.
- Tests that assert artifact existence but not semantic equivalence.

## Reject

- Recreating the historical three-repository topology.
- Backward-compatible preservation of ambiguous schemas and commands.
- A universal genotype combining every engine.
- Treating Linux fallback as scientifically equivalent without qualification.
- Treating pretraining or transferred priors as free in equal-budget claims.
- Keeping every engine indefinitely regardless of evidence.
- Requiring users to orchestrate separate research systems manually.

## Reframe

The predecessor platform was research-first. The successor is product-first: research discipline becomes internal assurance for returning a deployable model. Engines are not the user-facing product; the automatic portfolio, evidence, model selection, and verified bundle are.
